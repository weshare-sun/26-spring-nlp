\chapter{实验结果与分析}

本章基于第3章设计的三种 Word2Vec 实现方案，从训练效率、语义质量、可视化效果、超参数敏感性、语料规模影响和跨语言对比六个维度进行系统性实验评估。
所有实验在统一硬件环境下完成，确保对比的公平性。

\subsection{评估指标}

本实验采用以下评估指标：

\begin{itemize}
  \item \textbf{词相似度}：计算模型输出的余弦相似度与人工评分之间的 Spearman 相关系数。英文使用 35 对标准词对（WordSim-353 数据集），中文使用 40 对原生中文词对（非翻译版），避免翻译引入的语义偏移。
  \item \textbf{类比推理}：形式为 $a - b + c \approx d$。英文使用 77 组类比对，在经典 50 对基础上新增不规则动词、多义类比等难度层次，覆盖 12 个类别。中文使用 57 组原生中文类比（含省会-省份、作家-作品、动词体貌等中文特有类别）。
  \item \textbf{训练效率}：总训练时间和 Loss 收敛曲线。
  \item \textbf{可视化}：t-SNE 和 PCA 降维可视化，以及词对相似度热力图。
\end{itemize}

NumPy 版本耗时约 59 分钟，而 PyTorch 版本耗时约 83 分钟，这是因为 PyTorch 的自动微分机制在 CPU 模式下引入了额外的计算图构建和反向传播开销。

\subsection{Loss 收敛曲线}

图~\ref{fig:loss_curves} 展示了 NumPy 和 PyTorch 两个版本在训练过程中的 Loss 收敛情况。

\begin{figure}[htbp]
\centering
\includegraphics[width=0.85\textwidth]{figures/loss_curves.png}
\caption{NumPy 与 PyTorch 版本的 Loss 收敛曲线}
\label{fig:loss_curves}
\end{figure}

从收敛曲线可以观察到：
\begin{itemize}
  \item 两个版本的 Loss 均能稳定下降，最终收敛至 2.1--2.2 附近；
  \item PyTorch 版本的 Loss 曲线更加平滑，这归功于其自动微分机制带来的梯度计算稳定性；
  \item NumPy 版本由于采用手动实现的梯度更新，曲线略有波动，但整体趋势一致。
\end{itemize}

\subsection{效率差异原因分析}

\begin{enumerate}
  \item \textbf{NumPy 版本慢的原因}：其反向传播使用纯 Python 的 \texttt{for} 循环遍历每个训练样本进行梯度更新，Python 解释器的循环开销是主要瓶颈。
  \item \textbf{PyTorch 版本更慢的原因}：虽然 PyTorch 利用矩阵运算加速了前向传播，但在 CPU 模式下，自动微分机制需要构建和维护计算图，且 PyTorch 的张量操作在小批量 CPU 计算中未能充分发挥优势。
  \item \textbf{Gensim 版本极快的原因}：Gensim 底层使用 Cython 编写的优化代码，直接调用 C 语言级别的循环和内存操作，避免了 Python 解释器的开销。
\end{enumerate}

\subsection{类比推理评估}

类比推理是评估词向量的经典任务，形式为 \textit{a - b + c = ?}，例如 ``king - man + woman = queen''。

\section{超参数敏感性分析}

为探究超参数对词向量质量的影响，使用 Gensim 在 text8 子集（5000 句）上进行了三组消融实验，分别分析向量维度、窗口大小和负采样数的影响。

\subsection{向量维度}

固定窗口大小为 5、负采样数为 5，测试向量维度为 50、100、200 时的效果，结果如表~\ref{tab:ablation_dim} 所示。

\begin{table}[htbp]
\centering
\caption{向量维度消融实验}
\label{tab:ablation_dim}
\begin{tabular}{cccc}
\toprule
\textbf{维度} & \textbf{相似度相关} & \textbf{类比准确率} & \textbf{训练时间} \\
\midrule
50 & \textbf{0.4710} & 22.08\% & 9.2s \\
100 & 0.4352 & 23.38\% & 9.9s \\
200 & 0.4515 & \textbf{25.97\%} & 13.2s \\
\bottomrule
\end{tabular}
\end{table}

维度 200 在类比准确率上最优（25.97\%），但训练时间增加约 33\%。维度 50 在词相似度上表现最好（Spearman 0.4710），表明对于小规模语料，较低维度可减少过拟合。
综合考虑效率和质量，维度 100 是较优选择。

\subsection{窗口大小}

固定维度为 100、负采样数为 5，测试窗口大小为 3、5、10 时的效果，结果如表~\ref{tab:ablation_window} 所示。

这表明对于 Skip-gram 模型，较小的上下文窗口更适合捕捉精确的语法和语义关系。

固定维度为 100、窗口大小为 5，测试负采样数为 3、5、10 时的效果，结果如表~\ref{tab:ablation_neg} 所示。

\begin{itemize}
  \item \textbf{词表规模}：词表大小随语料增长呈对数增长趋势，从 10\% 的 19,461 词增长到 100\% 的 71,290 词；
  \item \textbf{词相似度}：30\% 语料时 Spearman 达到峰值 0.5790，之后略有下降但维持 0.52 以上，表明对于词相似度任务，中等规模语料已能取得良好效果；
  \item \textbf{类比准确率}：从 10\% 的 9.21\% 提升至 50\%--100\% 的 29.87\%，验证了``更多数据 = 更好的表示''这一规律，但 50\% 后收益递减；
  \item \textbf{训练时间}：与语料规模近似线性关系，100\% 的训练时间（36.8s）约为 10\%（3.1s）的 12 倍。
\end{itemize}

\section{实验总结}

\subsection{主要发现}

\begin{enumerate}
  \item \textbf{训练效率}：Gensim（Cython 优化）的训练速度是 NumPy 实现的 159 倍，是 PyTorch（CPU 模式）的 223 倍。纯 Python 循环和 CPU 模式下的自动微分机制是主要性能瓶颈。
  \item \textbf{语义质量}：三个英文模型在词相似度任务上表现相近（Spearman $\rho = 0.427$--$0.460$），Gensim 在类比推理（25.97\%）上略优。扩展至 77 对（含高难度类别）后，类比准确率更能反映真实语义推理能力。
  \item \textbf{超参数影响}：向量维度 100--200、窗口大小 3--5、负采样数 3--5 是推荐配置。更高的维度对类比推理有益但收益递减；较小的窗口和适中的负采样数在相似度上表现更优。
  \item \textbf{语料规模}：类比准确率随语料增长显著提升（10\%: 9.21\% $\to$ 100\%: 29.87\%），词相似度在 30\% 语料时达到峰值（Spearman 0.58），验证了数据规模对词向量质量的重要性，但存在收益递减。
  \item \textbf{可视化效果}：三种实现在 t-SNE 和 PCA 可视化中均展现出合理的语义聚类结构，验证了 Word2Vec 模型的有效性。
  \item \textbf{中文扩展}：全量中文维基百科（2,237,325 词）实验表明，Word2Vec 模型能够有效捕捉中文词汇的语义关系。全量模型 Spearman 相关系数达到 0.4145（逼近英文模型），类比准确率为 32.69\%，跨语言语义聚类在教育和科技类别上超越英文模型。
\end{enumerate}

\subsection{不足与改进方向}

\begin{enumerate}
  \item \textbf{GPU 加速}：PyTorch 版本当前仅在 CPU 上运行，未能利用 GPU 加速。Windows 平台下 AMD GPU 的 DirectML 后端存在算子覆盖不全的问题，训练无法收敛；未来可尝试在 Linux + ROCm 环境下启用 GPU 加速。
  \item \textbf{下游任务验证}：当前仅评估了词向量的内在质量（intrinsic evaluation），尚未在文本分类、命名实体识别等下游任务（extrinsic evaluation）上验证实际效果。
  \item \textbf{形态变化}：Word2Vec 模型在时态变化和比较级类比任务上表现较弱，这是其静态词向量表示的固有局限性。中文动词体貌类比（了/着/过）同样具有挑战性。
\end{enumerate}
